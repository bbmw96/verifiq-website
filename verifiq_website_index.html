Core Purpose
Two questions answered before every submission
VERIFIQ is built around the two checks every BIM coordinator needs to confirm before uploading to CORENET-X or NBeS.
1
Is the IFC model complete?
Is every element in the model populated with all the data that Singapore CORENET-X (IFC+SG 2025) and Malaysia NBeS (UBBL 1984) require? Checks entity classes, classifications, all Pset_ and SGPset_ property sets, property values, data types, georeferencing, and geometry — for all 8 Singapore agencies simultaneously.
2
Is the classification chain complete?
When a classification code is present on an element (e.g. A-WAL-EXW-01 for an external wall), are all the related SGPset_ property sets and their required property values also present? VERIFIQ checks the full classification-to-property chain for all 128 embedded IFC+SG codes, per agency, per CORENET-X gateway.

Workflow
From IFC to CORENET-X in 5 steps
1
Export IFC from your BIM tool
From ArchiCAD: use the IFC+SG Export Translator from info.corenet.gov.sg. From Revit: use the IFC+SG shared parameters and export settings. Export as IFC4 Reference View.
2
Open the IFC file in VERIFIQ
Click Open IFC File. VERIFIQ parses the STEP file natively — no xBIM, no cloud upload, fully offline. Supports .ifc, .ifczip, and .ifcxml.
3
Click Check All - Singapore
One click runs all 20 IFC data levels, all 128 classification code checks, and all 89 Singapore design code rules simultaneously. The Director's Report is generated automatically.
4
Fix issues with the Property Editor
Click Fix on any Critical or Error finding. Enter the correct value. Click Apply Fixes. VERIFIQ saves a corrected IFC file — no need to return to ArchiCAD or Revit for property-level fixes.
5
Export report and submit
Export a PDF compliance report, BCF file for your BIM team, and Excel findings workbook. Submit the corrected IFC file to CORENET-X with confidence.
