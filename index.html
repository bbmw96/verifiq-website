Core Purpose

Two questions answered before every submission

VERIFIQ is built around the two checks every BIM coordinator must confirm before uploading to CORENET-X or NBeS.

1

Is the IFC model complete?

Is every element populated with all the data that Singapore CORENET-X (IFC+SG COP 3.1, December 2025) and Malaysia NBeS (UBBL 1984) require? VERIFIQ checks entity classes, classifications, all Pset_ and SGPset_ property sets, property values, data types, enumerations, spatial containment, georeferencing (SVY21/GDM2000), GUID uniqueness and geometry validity for all 9 Singapore agencies simultaneously.

20 check levels 9 agencies SG + MY
2

Is the classification chain complete?

When a classification code is present on an element (for example, A-WAL-EXW-01 for an external wall), are all the related SGPset_ property sets and required property values also present? VERIFIQ checks the full classification-to-property chain for all 196 embedded IFC+SG COP 3.1 codes, per agency, per CORENET-X gateway.

196 codes SGPset_ chain Per gateway
How classification drives IFC in ArchiCAD and Revit: In ArchiCAD, elements are not defined by IFC entity class directly. A slab element can be exported as IfcWall as long as the classification code is set to a wall code. The selected IFC+SG translator maps the classification to the correct IFC entity class and retrieves all linked property sets and required properties. Each element has exactly one classification, which drives its IFC class, its property sets and all required property values. VERIFIQ validates this entire chain.

Geometry Reference

Geometry reference only. OBJ, FBX, STL and STEP files are opened as visual reference alongside the IFC model. VERIFIQ provides guidance on how to export a compliant IFC file from the originating application. Compliance checking requires an IFC file.

Visual reference only
.e57   .las   .pdf   .json

Reference Data

Point cloud (.e57, .las), PDF drawings and JSON data files are opened as reference documents alongside the IFC model. PDF drawings can be viewed in the same session as the 3D compliance overlay for as-built verification. Compliance checking requires an IFC file.

Reference only
Note on ArchiCAD and IFC classification: In ArchiCAD, elements are not defined by IFC entity class directly. A slab element can be exported as IfcWall as long as the classification is set to a wall classification code in ArchiCAD. The selected translator then maps the classification code to the correct IFC entity class (IfcWall) and retrieves the linked property sets and required properties for that class. Each element has exactly one classification, which drives its IFC entity class, its property sets and all required property values. VERIFIQ validates this entire chain.
